# 课程信息

## 课程名称

uv 包管理工具

## 课程别名

用户可能会这样称呼这门课：

- uv
- uv 教程
- uv 学习
- Python uv

## 课程目标

- 掌握 uv 作为统一 Python 工程工具的核心用法
- 理解 uv 能替代哪些工具及其设计思路
- 能独立使用 uv 管理项目依赖、Python 版本、虚拟环境和全局工具

## 学习主线

从 Python 工程化的痛点出发，逐步掌握 uv 统一工具链：安装 → 项目管理 → 全局工具 → Python 版本管理 → 脚本 → 工作区 → 缓存与容器。

## 章节安排

章节划分应与 `reading_plan.md` 的分片计划对齐。每个章节对应一个或多个片段。

1. uv 是什么 + pip 替代（快速安装依赖）
2. poetry 替代（项目管理与锁文件）
3. pipx 替代（全局 CLI 工具）
4. pyenv + virtualenv 替代（Python 版本与虚拟环境管理）
5. 脚本运行（内联依赖元数据）
6. 工作区支持（Cargo 风格 Monorepo）
7. 全局缓存与容器化
8. PyTorch 适配 + conda 迁移

## 当前优先级

- 第 1 章最优先：先理解 uv 的定位和最基本用法

## 可跳过或延后的内容

- PyTorch 适配和 conda 迁移章节如学习者无相关需求可延后

## 适合的授课方式

- 默认使用灵进行苏格拉底式教学。
- 优先从"Python 工程化的痛点"和具体使用场景进入概念。
