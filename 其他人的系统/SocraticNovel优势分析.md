## 架构对比

### SocraticNovel 四层架构

```
启动层（copilot-instructions.md）→ 加载时序控制
教学层（system_core.md + knowledge_points/ + curriculum.md）
叙事层（story.md + story_progression/ + characters/）
运行时层（progress + session_log + review_queue + mistake_log + diary + wechat_group）
```

### 伙伴共学系统架构

```
路由层（CLAUDE.md + map.py）→ 场景分发
内核层（system_detail.md + classroom.md）
角色层（ling.yaml + xia.yaml）
课程层（courses/<课程名>/ 下各文件）
```

**核心差异**：SocraticNovel 有独立的叙事层和更丰富的运行时层，伙伴共学系统的角色和教学耦合更紧。

## 值得借鉴的 8 个优势

### 1. B+C 防模式切换机制

**是什么**：LLM 最危险的失败模式——学生说了接近答案的话 → AI 训练触发"确认+展开"→ 从观察者切换成传输者。B+C 是对此的防护：

- **B（思考顺序）**：先分析学生上一句话 → 定位他在知识地图的哪个位置 → 然后才看教案里程碑
- **C（盲测）**：*"如果我没听到学生的话，我还会问同样的问题吗？"* → YES = 铁路问题（按教案走而非回应学生），必须重写

**对伙伴共学系统的价值**：灵的现有自检门缺少对"按预设轨道走"的检测。盲测 C 可以直接加为第 5 项自检。

---

### 2. 三铁律 + 最后一英里规则

**三铁律**（永远在工作记忆中）：

| 铁律 | 含义 | 违反信号 |
|:---|:---|:---|
| 只问不说 | 每回复问句 ≥ 陈述句 | 发现自己在"解释一个概念" |
| 眼睛看学生 | 每个问题从学生上一句话里长出来 | 学生没提到 X，你却在问关于 X 的问题 |
| 答案是学生的 | 术语、结论、命名——全部由学生说出口 | 你替学生总结了、命名了、下结论了 |

**最后一英里规则**：当学生离答案 ≤1 步时 → 不确认、不否认、不总结 → 追问"你怎么理解你刚用的那个词？"

**对伙伴共学系统的价值**：灵的现有规则偏向"不能做什么"，三铁律给出了"应该做到什么"的正向锚点。最后一英里规则补充了灵在对方快推出来时最容易崩塌的场景。

---

### 3. 五级支持梯度 S1-S5

学生卡住时的降级路径，**不跳级**，每级至少试一次：

| 级别 | 方式 | 示例 |
|:---|:---|:---|
| S5 | 沉默等待 | "你再想想？" |
| S4 | 缩小范围 | "你觉得答案会比 X 大还是小？" |
| S3 | 提供类比（先问学生有没有） | "就像水总是往低处流……" |
| S2 | 给出半个线索 | "如果我们只看其中一条线呢？" |
| S1 | 方向性事实（⚠️ 绝不给结论） | "有限的规则不一定对无穷成立" |

S1 之后仍卡住 → 切到更基础的前置知识，标记"回头补"。

**对伙伴共学系统的价值**：当前只有"给半步 → 还卡 → 标记回头补"，粒度太粗。S5→S1 给了灵一个渐进的工具箱。

---

### 4. P0-P5 结构化教学阶段

```
P0 课前设计 → P1 侦察起点 → P2 先行失败 → P3 矛盾暴露 → P4 苏格拉底重构 → P5 命名与回看
```

**P0 悖论**（关键设计）：P0 产出的是**地图**，不是**剧本**。禁止预设问题序列、类比、if-then 分支。P1 侦察后根据实际状态现场调整。

**P2 先行失败**（有论文支撑）：让学生先用自己的直觉回答一个问题——不管对错。Kapur (2008, 2024) 的 Productive Failure 研究证明：先尝试、先失败、再引导，比直接教效果更好。

**对伙伴共学系统的价值**：P2（先行失败）是现有流程完全缺失的。伙伴共学系统的"自然讨论"假设学生能直接参与，但缺少刻意的"先暴露错误模型"环节。

---

### 5. 学生类型预案

6 种学生类型的实时识别 + 对应策略：

| 类型 | 信号 | 应对 |
|:---|:---|:---|
| 白板学生 | 频繁"不知道" | S5→S1 逐级降级；退一步确认前置知识 |
| 过度自信 | 自信地答错 | P3 矛盾暴露；追到推论荒谬处 |
| 跳跃学生 | 突然说出答案 | 最后一英里规则；追问"你怎么理解这个词？" |
| 沮丧学生 | "直接告诉我" | 承认感受 → 指出已知的 → 用简单问题重建信心 |
| 跑题学生 | 偏离话题 | 引用他之前说的话做桥梁拉回 |
| 反问学生 | 反过来问你 | 拆解为子问题反抛回去 |

**对伙伴共学系统的价值**：当前 learner_profile 是静态画像，缺少课堂中的动态识别。可作为灵在 thinking 中的判断工具。

### 6. 写作红线与情绪工具箱

**四条禁令**（每一条都是踩坑踩出来的）：
- 禁止 `*斜体动作*`（*微微一笑*）——这是舞台指示，不是描写
- 禁止 `[方括号动作]`（[叹气]）——这是剧本标注
- 禁止空洞情绪形容词——不写"她很感动"，写她做了什么
- 禁止连续两轮同类型描写

**情绪工具箱**（8 种不说"我很感动"的方法）：动作替代、节奏变化、环境折射、物品聚焦、沉默、微动作、距离变化、中断。

**核心原则**：关系变化不靠"态度变好了"告诉读者，靠行为层面的微观变化让读者自己推理——"她回答时多停了一拍确认你跟上了——以前她不会等。"

**对伙伴共学系统的价值**：当前的角色呈现偏向正向指导（"动作要具象""要有口语颗粒感"），缺少负面禁令和替代方案。情绪工具箱可以直接复用。

---

### 7. 运行时状态管理丰富度

SocraticNovel 的运行时层比伙伴共学系统多了：

| 文件 | 用途 | 伙伴共学系统对应 |
|:---|:---|:---|
| `review_queue.md` | 间隔复习队列（知识点级别调度） | 无系统化调度，靠手动标记 |
| `mistake_log.md` | 结构化错题本 | 无 |
| `wechat_group.md` | 角色群聊完整记录 | 课后群聊存在但不持久化展示 |
| `wechat_unread.md` | 待查看未读消息（下次上课的钩子） | 无 |
| `diary.md` | 学习日记（含写作规则） | 无 |
| `session_log.md` + `session_archive.md` | 课堂日志 + 压缩归档 | 有 progress.md 但不分层 |

**wechat_unread 机制特别值得借鉴**：制造了"下次还想打开"的心理锚点——不是教学需要，是连接理由。

---

### 8. META_PROMPT 生成框架

允许用户通过回答 AI 的交互式问题，从零生成一整套 SocraticNovel 系统：
- Phase 1：角色创建（姓名/性格/教学风格/核心矛盾）
- Phase 2：世界观构建（地点/特殊性/关系）
- Phase 3：课程结构（章节/知识点/练习册）
- Phase 4：故事线设计（情感阶段/必然事件/暗线种子）
- Phase 5：生成完整文件系统

**对伙伴共学系统的价值**：如果未来想让其他人也能快速搭建自己的共学系统，META_PROMPT 模式可以参考。当前阶段不需要。

## 关键设计哲学差异（需要注意的）

1. **SocraticNovel 是"老师教学生"模式，伙伴共学是"三人平等"模式。** 三铁律的"只问不说"在伙伴共学中需要调整——灵和夏也是学习者，她们也可以表达自己的困惑和发现。不能照搬"绝不直接给答案"——灵可以分享自己的推理过程，只要不替代学习者走最后一步。

2. **SocraticNovel 有完整剧情线（情感四阶段、暗线种子回收），伙伴共学不需要。** 伙伴共学的关系推进是"有机的"——通过一次次共学自然积累。刻意设计剧情线会破坏真实感。

3. **SocraticNovel 的 teacher 角色会"表演"教学风格（凛的隐喻空间、律的冷幽默），灵和夏是"活人"但不是演员。** 借鉴写作技巧（如何用行为表达情感）而非借鉴角色设定。
