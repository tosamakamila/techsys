# 课后自适应更新模板

每节课结束后，按本模板更新长期记忆。目标是让下一节课更贴合学习者，而不是保存流水账。

如果本节课属于 `courses/` 下某门具体课程，课程进度、课堂状态、教材修订和会话归档必须写回该课程文件夹；`teacher/` 只保存跨课程通用的教学系统、角色设定和学习者长期画像。

课堂中不要提及这些更新动作。只有当用户说“下课”“今天到这里”“做课后更新”等表达后，才进入维护状态。

## 更新总原则

1. 只记录会影响后续教学的内容。
2. 区分“一次性表现”和“稳定特征”。
3. 对学习者的判断要保守，避免因为一节课就贴标签。
4. 记录具体证据，而不是空泛评价。
5. 下一课入口必须清晰，让老师下次能自然继续。

## 课后更新流程

### 1. 先做课堂诊断

先在心里回答以下问题，再写入文件：

- 今天学了什么？
- 学习者亲自推出来了什么？
- 哪一步最明显卡住？
- 哪类提示最有效？
- 哪个误区值得下次复习？
- 本节课暴露的是一次性问题，还是可能的长期薄弱点？
- 下一课应该继续推进、复习巩固，还是换例子迁移？

### 2. 更新当前课程的 `progress.md`

记录学习进度、本课摘要和历史归档。建议格式：

```markdown
## 最近一课

- 日期：
- 主题：
- 教材/材料：
- 教材片段：
- 完成情况：
- 本课标签：
- 本课摘要：（2-3 句话）
- 待复习：
  - 
```

同时将此次”最近一课”追加到”历史归档”中，保留片段ID和简短摘要即可。

注意：

- “完成情况”要具体，例如”完成注意力机制必要性与 Q/K/V 直觉，尚未进入多头注意力”。
- “本课摘要”写 2-3 句话：推理主线 + 关键顿悟 + 未解决点。
- 学生推出来的结论统一记录在 `lesson_state.md` 的”已建立的结论”中，此处不再重复。
- “待复习”写下一次最该补的 1-3 个点。

### 3. 更新当前课程的 `lesson_state.md`

这是下一次开课最重要的文件。它同时包含下节课的教案入口和持久学习状态。

建议格式：

```markdown
## 本节课

- 课程类型：新课 / 复习课
- 片段ID：
- 教案文件：
- 原文范围：

### 本堂课结束点

- 本堂课只处理：
- 不处理：
- 下课条件：

### 本课标签

- 
- 

## 已建立的结论

- 

## 正在卡住的问题

- 

## 下次开课建议

- 

## 下一次对话入口

灵可以这样开始：
> 上次我们已经推出了……今天先从一个小问题继续：……
```

要求：

- “下一次对话入口”必须写成自然课堂语言。
- 不要写“读取系统后继续”之类幕后表达。
- 如果本节课完整结束，就写清下一课主题。
- 如果本节课中断，就写清中断位置和下一问。

### 4. 更新当前课程的 `reading_plan.md`

每节课结束后，必须推进教材的读取位置，标记已完成的片段。

更新内容：

- 将本节课片段的"状态"更新为"已上课"（或"需复习"）。
- 在"已完成标签"中追加本课简短标签。
- 更新"当前读取位置"为下一个片段。
- 如果有必要回顾的旧片段，在"需要回顾的旧片段"中注明。

格式参考：

```markdown
## 当前读取位置

- 当前教材：M1
- 下一个片段：L003
- 前置依赖：需先回顾 L001 的核心概念
- 需要回顾的旧片段：L001
```

**如果是复习课**，更新方式不同：

- 卡点消除 → 将复习的旧片段状态从"需复习"改为"已上课"，当前读取位置保持原推进目标。
- 仍卡住 → 保持"需复习"，在 `lesson_state.md` 中更新卡点描述（换策略、换例子），下次再试。
- 不推进到新片段。

### 5. 更新 `learner_profile.md`

采用**记忆压缩**机制，不逐课堆砌观察。

**写入条件：** 同一模式出现 ≥2 次时才写入。

**压缩规则：**

| 出现次数 | 行为 |
|----------|------|
| 首次 | 不写入。在心里记下。 |
| 第 2 次（同一模式） | 写入”观察证据”：日期 + 现象 + 可能含义 |
| 第 3-4 次（同一模式确认） | 压缩为”特征”，写入对应栏目（薄弱点/有效策略/需避免）。删除原始”观察证据”条目 |
| ≥5 次 | 标记为”稳定特征”，上课时默认应用 |

**栏目：**

```markdown
## 学习偏好

- 

## 已知薄弱点

- 

## 有效教学策略

- 

## 需要避免的教学方式

- 

## 观察证据

- 日期：现象：可能含义：
- 日期：现象：可能含义：（第 2 次出现，与上条同一模式，即将压缩）
```

写法要求：
- 用”可能””倾向于”而不是绝对判断。
- 压缩后的特征用肯定句式（如”抽象概念首次接触时需从具体例子入手”）。
- 观察证据保留最近 3 条，更早的压缩后删除。
- 保留证据，例如“在卷积课中，对空间共享直觉理解快，但对通道维度容易混淆”。
- 不记录与学习无关的隐私或情绪细节。

### 6. 更新当前课程的 `book_revision_notes.md`

如果教材暴露出问题，记录修订建议。

建议格式：

```markdown
## 日期 - 主题

- 来源材料：
- 暴露的问题：
- 学习者卡住的位置：
- 建议补充：
- 建议删减或调整：
- 为什么这会帮助后续学习：
- 建议制卡：是/否
- 制卡原因：（课堂中在这个点上卡住了 / 教材没有但考试会考）
- 教师补充内容：（教材缺失的知识，由老师直接撰写；简明扼要，后续上课可作为补充材料）
```

下课时，扫描 book_revision_notes.md 中"建议制卡=是"的条目，一并纳入 card_material.md 标记清单。

### 7. 更新闪卡原材料 `card/<课程名>/card_material.md`（询问后触发）

**重要**：此步骤必须在下课流程的"制卡询问"环节之后，且学生同意时才执行。学生拒绝则跳过。

card_material.md 不是完整闪卡内容，而是一份**标记清单**——只记录"哪些知识点建议做成闪卡"和"为什么"，不写完整答案。

标记格式：

```markdown
## 标记：<知识点名称>
- 为什么值得制卡：（学生在课堂上在这里卡住了 / 容易混淆 / 考试高频）
- 关键得分点（≤3条）：
  - 
- 常见错误：
  - 
- 来源：教材第X章 / 老师补充
```

写入规则：
- 只从 lesson_state.md 的结论和课堂中暴露的问题中筛选。
- 只标记值得制卡的，不是把所有结论都标上。
- 每条标记控制 3 条以内关键得分点。
- 如果该知识点学生尚未稳定掌握，暂不标记。
- 如果标记已在之前的 card_material.md 中，不要重复写入。
- 同时扫描 book_revision_notes.md 中"建议制卡=是"的条目，一并纳入标记清单。

### 8. 更新 `courses/<课程名>/diary.md`（可选）

仅当用户希望保留学习体验时更新。如果文件不存在则创建。

日记可以更生活化，但仍要克制。可以包含灵、夏与学习者的课堂氛围，但不要让日记喧宾夺主。

### 9. 可选：更新角色状态

只有当课堂互动明显影响角色关系或课堂风格时，才更新角色文件。

例如：

- 夏因为某次共同卡住而更愿意提前预习。
- 灵发现学习者更适合先用反例再给定义。

不要每节课都更新角色人格。角色人格应稳定，变化要慢。

## 更新后的自检

更新完成后，检查：

- 下次说”开始上课”时，老师能否自然接上？
- `reading_plan.md` 的读取位置是否已推进到下一个片段？
- 本节课片段的完成标签是否已写入 `reading_plan.md`？
- 是否记录了学习者真实卡点？
- 是否避免了过度贴标签？
- 是否保留了会影响教学的证据？
- 是否没有在课堂语言里暴露系统、文件、模块等幕后信息？
