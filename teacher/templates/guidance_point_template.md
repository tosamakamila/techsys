# 引导点预案模板

在教材原文（`materials/`）对应位置，为核心概念嵌入引导点标注。一节课不超过 3 个。HTML 注释是机器可读的结构标记，正文是引导者的预案参考。（引导点直接标在教材原文中，不需要 transformed/ 教案。）

## 格式

```markdown
## 引导点: [概念名]

<!-- gp: Nxxx -->              ← 对应 knowledge_map 节点ID（必填）
<!-- gp: prereq Nxxx -->       ← 前置知识节点（可选，多个则多行）
<!-- gp: type 对比引导 -->     ← 对比引导/过程拆解/推到极端/反例检验/场景假设（必填）

**铺路：** [1-2句，总结已建立的前提。把缺口的方向指清楚]

**主问题：** [开放引导问题。不能是是非题。必须有推理支点]

**预期回答分支：**

→ ✅ 方向正确（[信号：比如"提到分裂图像/否定了自然发生"]）：
   "[引导者的回应——顺着答案推下一步，不评分]"

→ ⚠️ 部分正确（[信号：比如"说了分裂但没排除其他可能"]）：
   "[引导者的回应——指出缺口方向，给半步]"

→ ❌ 方向偏离（[信号：比如"回答细胞怎么分裂的细节"]）：
   "[引导者的回应——笑着拉回，换角度重新框定]"

**过渡：** [无论哪条路径，最终桥接到下一个概念。1句话]
```

## 示例

```markdown
## 引导点: [概念名]

<!-- gp: Nxxx -->
<!-- gp: prereq Nxxx -->
<!-- gp: type [对比引导/过程拆解/推到极端/反例检验/场景假设] -->

**铺路：** [已建立的已知条件A和B。指出缺口方向——从"是什么"转向"从哪里来/为什么"]

**主问题：** [开放引导问题，从具体观察出发，含推理支点]

**预期回答分支：**

→ ✅ 方向正确（[信号描述]）：
   "[引导者的回应——顺着推下一步，不评分]"

→ ⚠️ 部分正确（[信号描述]）：
   "[引导者的回应——指出缺口方向，给半步]"

→ ❌ 方向偏离（[信号描述]）：
   "[引导者的回应——拉回主线，换角度重新框定]"

**过渡：** [桥接到下一个概念，1句话]
```

## 使用规则

1. **预案不是剧本。** 引导者上课时不逐字读——这是备课时存入的判断框架
2. 学习者回答不在三个分支中 → 引导者按 teacher/system_detail.md「共学内核」节引导法则临场应对
3. 一节课不超过 3 个引导点——只标最关键的概念转折处
4. 引导者开口仍是口语化、带颗粒感的，不是教案腔
5. 引导点由备课阶段填充，上课前检查：教案有 `<!-- gp: -->` 则参考，无则引导者临场引导
