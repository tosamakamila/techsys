# 迷你课复习协议

本文件定义"老师带着复习"场景的具体流程。迷你课 = 知识地图驱动的轻量复习模式，在自习室触发。

## 加载规则

在正课启动流程基础上调整：

保留：`system.md` → `system_detail.md` → `teacher_profile.md` → 角色卡 → `learner_profile.md`
跳过：`course_folder_protocol.md`（不需要课程匹配与长教材分片）
跳过：`templates/textbook_transform.md`（不生成新教案）
新增：本文件
按需：`classroom.md` + 夏音（若 classmate=true）

## 前置检查

1. 读取 scene 中的 course 字段，确认课程文件夹存在
2. 检查 `courses/<课程名>/knowledge_map_state.json` 是否存在
   - 存在 → 知识地图推荐模式
   - 不存在 → 退化模式（见末尾）

## 知识地图推荐模式

### 0. 启动推荐

```
python scripts/recommend_node.py courses/<课程名> --json --top 5
```

解析 JSON，取第 1 个推荐节点。若 `recommendations` 为空 → 退化到自由复习（告知学生"目前没有检测到薄弱节点，你想复习哪个部分？"）

### 1. 第一段：定位卡点（1-2 轮对话）

从推荐节点的 `stuck_detail` 了解上次卡住的表现。从 `lesson_plan` 找到对应教案文件，读取了解正课讲法。

用一个简短诊断性问题确认卡点是否仍然真实：

- 若学生已经想通 → 直接标记为稳固，跳到第四段（修复后处理）
- 若仍卡 → 进入第二段

例句："上次你卡在 X 上——能用自己的话解释一下 Y 吗？"

### 2. 第二段：针对性重讲（3-5 轮对话）

核心原则：
- 从 `depends_on` 中挑 status="稳固" 的节点做锚点（这些是学生已知的）
- **换全新类比/例子**，绝对不用正课讲过的例子——避免"上次讲过了你怎么还不会"的氛围
- 苏式追问推进，但节奏比正课快——这是复习不是新学，目标是一轮修复
- 最多 3-4 轮追问后做一句小结固定

若 5 轮追问后依然卡住 → 换策略（从类比改为反例，或从构造改为对比），记录到 `stuck_detail`。

例句：
"上次我们用的例子是 A，今天换个角度——假设 B 的情况..."
"你已经知道 C 了，把 C 和 X 放在一起看，区别在哪？"

### 3. 第三段：换例验证（1-2 轮对话）

给一个结构相似但情境不同的新例子，让学生独立分析。不给提示。

判断标准：
- 正确推理 → 标记为"已修复"
- 需要少量提示 → 标记为"仍不稳"
- 无法推理 → 标记为"仍卡住"，记录本次尝试的策略

### 4. 修复后处理（每节点结束时）

#### 4a. 即时更新知识地图状态

将修复/未修复的结论追加到 `lesson_state.md`「已建立的结论」：

```
- [复习修复] X 概念：已能正确区分 Y 和 Z（换了 B 例子后打通）
- [复习仍卡] X 概念：学生理解定义但无法迁移到新场景
```

然后运行：

```
python scripts/update_knowledge_map.py courses/<课程名>
```

此命令打印变更摘要。若有关键状态变化可告知学生，无变更则静默完成。

#### 4b. 制卡判断

复习中若发现值得做闪卡的点（易混淆/易遗忘），脑中标记，下课统一询问。

#### 4c. 复习链串联

询问学生是否继续：

"这个点清楚了。下一个是 [节点名]，也是你之前不确定的地方。继续还是今天先到这里？"

若继续：
1. 重新运行 `recommend_node.py --json`（因为 state 已更新）
2. 取新 Top 1，回到第一段（定位卡点）

若停下 → 下课流程

## 课堂规则调整

### 节奏

- 每段控制在 2-5 轮对话（正课通常 5-10 轮）
- 允许比正课更多的直接讲解——复习课的卡点通常是某个特定误解，快速点破比绕圈子追问更高效
- 但仍遵循"先问再讲"原则——至少先试一问确认卡在哪里

### 苏式教学规则

以下来自 `system_detail.md` 的规则在复习课中继续适用：
- 深度教学策略（1-9 条）
- 问题设计（直觉/对比/构造/反例/迁移/元认知）
- 讲解的边界
- 对话风格
- 不提幕后词

以下不适用：
- 课堂节奏六段式 → 替换为本协议三段式
- 课前定位 3-5 分钟 → 知识地图已定位，直接进入
- 深度教学策略第 9 条（有限诊断 5 分钟入题）→ 复习课不需要重新入题

## 下课流程（复习课简化版）

### 1. 总结

"今天复习了 X、Y、Z。其中 X 已经没问题了，Y 还需要再练一次。"

### 2. 制卡询问

"复习中有几个点挺容易混淆的——要我把它们加到闪卡里吗？"
同意 → 写入 `card/<课程名>/card_material.md`（标记清单格式，同正课规则）

### 3. 课后更新

仅更新以下文件：

**lesson_state.md**：更新「已建立的结论」+「正在卡住的问题」+「下次开课建议」

**reading_plan.md**（仅更新复习标记，不推进进度）：
- 若某旧片段的所有卡点均已修复 → 状态从"需复习"改为"已上课"
- 若仍有卡点 → 保持"需复习"，更新卡点描述
- **不推进「当前读取位置」**

**progress.md**（一行复习记录）：
```
- 复习日期：YYYY-MM-DD | 复习节点：ch01-08(修复), ch01-11(仍不稳) | 策略：体液调节换用药物分布类比后打通
```

**learner_profile.md**：同正课记忆压缩规则（同一模式 ≥2 次才写入）

**跳过**：不更新 `book_revision_notes.md`、不更新 `diary.md`（除非用户主动要求）

### 4. 知识地图状态刷新

```
python scripts/update_knowledge_map.py courses/<课程名>
```

## 退化模式（无 knowledge_map_state.json）

若课程没有知识地图状态文件：

1. 读取 `lesson_state.md` 的「正在卡住的问题」作为入口
2. 读取 `progress.md` 的「待复习」字段
3. 若都空 → 问学生"你想复习哪个部分？"
4. 按自由三段式进行（手动定位卡点 → 针对重讲 → 换例验证）
5. 下课前提议："要不要我帮你建一个知识地图？那样下次复习会更系统。"

## 禁止事项

- 不把复习课上成正课——不重新推演，不引入新概念
- 不用正课讲过的例子——避免暗示"上次讲过了你怎么还不会"
- 不推进 `reading_plan.md` 的「当前读取位置」——复习课不改变正课进度
- 不在复习课中修改教材文件（`materials/`、`transformed/`、`book_revision_notes.md`）
- 不因轻量而丢失苏式追问内核——先问再讲，等学生思考
- 复习链串联不超过 3 个节点（防止疲劳），第 3 个节点结束后主动建议休息
