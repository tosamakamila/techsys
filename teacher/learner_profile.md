# 学习者画像

这个文件记录会长期影响教学方式的信息。不要把每次课堂的琐碎表现都写进来。

## 背景

- 专业：生物
- 兴趣方向：计算机、深度学习
- Python 基础：薄弱。会用脚本但不太理解虚拟环境、包管理机制。
- 之前的工具链：conda（习惯 base 环境）→ scoop 安装 uv
- 正在学习的主线：uv 包管理工具

## 学习偏好

- 偏好通过提问、追问和亲自推理来学习。
- 对"为什么需要这个概念"比较敏感。
- 喜欢动手实操，体验过工具的速度后会更有动力。
- 坦诚自己不懂的地方，不装懂。

## 已知薄弱点

- Python 工程化基础：虚拟环境、包管理、环境隔离概念需要强化。
- conda 和 uv 的对应关系容易混淆。

## 有效教学策略

- 从具体例子和动手操作进入抽象概念。
- 先让学生亲手跑通，再解释背后原理。
- 控制每节课的新概念数量，一次只钉牢 2-3 个点。
- 学生卡住时直接帮搭桥（如 mkdir + uv venv），不让学生困在环境问题上。

## 需要避免的教学方式

- 不要一次给太多新概念，容易混淆。
- 不要在基础环境问题上反复追问——学生基础薄弱时容易造成挫败。
- 不要用"conda 你应该知道"之类假设前提的讲法。

## 观察证据

- 2026-05-19 uv 第一课：学生对虚拟环境概念不熟，但动手操作后理解明显提升。提示"先建 uv-test 文件夹"这类简单指令比解释概念更有效。

## 待观察

- 进入系统化项目管理（uv init/add/run）后是否容易跟上。
- 是否需要更多图示辅助理解。
