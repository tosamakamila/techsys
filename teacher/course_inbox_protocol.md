# 课程教材投递箱协议

`course_inbox/` 是预存课程教材的临时入口。用户可以把教材先放在这里，教材名对应课程名。

当用户说“上某某课”时，Claude Code 应先检查 `courses/` 中是否已有该课程；如果没有，再检查 `course_inbox/` 中是否有匹配教材。

## 匹配规则

假设用户说：

> 上机器学习课

匹配顺序：

1. 在 `courses/` 下寻找已存在的课程文件夹。
2. 如果没有，检查 `course_inbox/` 下是否有文件名包含“机器学习”的教材。
3. 如果有多个匹配文件，把它们视为同一课程的候选教材。
4. 如果匹配不确定，简短询问用户确认。

## 自动建课流程

如果 `courses/` 下没有对应课程，但 `course_inbox/` 中有匹配教材：

1. 根据课程名创建课程文件夹。
2. 从 `courses/_template/` 复制基础结构。
3. 将匹配教材移动到新课程的 `materials/` 文件夹。
4. 投递箱中不再保留该教材原文件。
5. 更新新课程的 `course.md`：
   - 课程名称
   - 课程别名
   - 教材来源
   - 当前课程状态
6. 更新新课程的 `progress.md` 和 `lesson_state.md`，说明课程已创建、教材已导入。
7. 如果教材很长（超过一两万字），初始化 `reading_plan.md`：
   - 识别教材结构（目录、章节标题）。
   - 建立教材列表和分片计划。
   - 设置第一个片段的当前读取位置。
8. 询问用户是否要先按 `teacher/templates/textbook_transform.md` 为第一个片段生成教案，还是直接开始诊断式上课。

## 关于删除原教材

“删除原教材”在这里指：教材不再留在 `course_inbox/` 中。

实际操作应优先使用“移动”而不是复制后删除：

- 从 `course_inbox/教材文件`
- 移动到 `courses/课程名/materials/教材文件`

这样既完成清空投递箱，又避免误删。

## 课后更新

课程一旦创建，后续所有课程状态都写入该课程文件夹，不再写入 `course_inbox/`。
